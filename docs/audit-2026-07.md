# Deep audit — July 2026

Full-repository audit of the design system, run July 2026 against the 1.0.0 pin. This document records the verdict, the method, every finding by severity, and what the follow-up hardening change fixed versus deferred. The companion benchmark reference is `docs/quality-gates.md`.

## Verdict

**B+ / launch-viable with fixes.**

Rare strengths: 99/99 primary components ship the full contract trio (`.jsx` + `.d.ts` + `.prompt.md`), a 22-gate deterministic fast board runs in CI, the token pipeline spans CSS → JSON/ESM → W3C DTCG → native (SwiftUI/Compose/Flutter) with provenance, and all 84 templates are bilingual with whole-set overflow audits across responsive/language/theme.

Main risks found: cross-format token drift (a proven bug, not a hypothesis), documentation count drift, bundle regeneration only possible inside Claude Design sessions, and Storybook thin as a documentation hub.

## Method

Five parallel deep dives, each covering its whole surface rather than sampling (per the CLAUDE.md verification-depth rule):

1. **Tokens & components** — every `tokens/*.css` value cross-checked against `tokens.json`, `tokens.js`, `tokens.dtcg.json`, and `tokens/native/*`; every component `.jsx`/`.d.ts`/`.prompt.md` read for contract drift, a11y gaps, and dead props.
2. **Templates, guidelines & UI kits** — all 84 templates probed for bilingual leaks, stale enums, and schema-sidecar coverage; guideline pages and the three UI kits checked against doctrine.
3. **Gates & CI** — every `_audit/*.html` gate's assertion code read; the GitHub Actions workflow compared against the gate inventory and `package.json` scripts.
4. **Docs & agent surfaces** — README, SKILL.md, llms.txt, CONTRIBUTING, and every `docs/*.md` checked claim-by-claim against disk and `_ds_manifest.json`.
5. **Storybook & distribution** — the Storybook config, story inventory, `package.json` packaging fields, `_esm/` entry, and tracked-artifact hygiene.

## Findings by severity

### Critical (breaks consumers or trust)

- **Motion token drift across formats.** `--cs-duration-instant/fast/base/slow` are 80/120/200/320ms in `tokens/motion.css` but **0ms** in `tokens/tokens.json`, `tokens/tokens.js`, `tokens/tokens.dtcg.json`, and all `tokens/native/*` targets — the generator ran under a reduced-motion condition and baked the collapsed values in. No CSS↔JSON parity gate existed to catch it.
- **Unit tests never ran in CI.** `npm run test:unit` (6 contract tests, including Storybook coverage) was absent from `.github/workflows/design-system-gates.yml`.
- **Packaging hole.** `package.json` `files` omitted `_esm/` while `exports["."]` points at `_esm/cs.mjs` — a pack would have dropped the ESM entry.

### High (drift / correctness)

- **Doc drift** (all corrected in this change):
  - `README.md` overstated the Employment Suite as 41 documents — the real count is 37 `vn-*` instruments.
  - `README.md` "Sources" cited a `DESIGN.md` file as if in-tree — the doctrine file lives in the upstream repo, and `@cyberskill/tokens|react|brand-assets` are external upstream packages.
  - `README.md` implied all 115 exports carry the `.jsx`/`.d.ts`/`.prompt.md` triplet — the 99 primaries do; 16 co-exports are documented in their parents' files.
  - `README.md` functional group bullets listed only a subset of what ships (e.g. Forms showed ~11 of 27 primaries; Overlays 2 of 6).
  - `CONTRIBUTING.md` said DatePicker/Combobox were out of scope — both ship as full components.
  - `docs/template-schema-v2.md` said 21 sidecars — the real count is 26 (`ls templates/*/content-schema.json`).
  - Three prompt files (`components/data/Card.prompt.md`, `components/feedback/Alert.prompt.md`, `components/data/Timeline.prompt.md`) carried example copy citing 50 legacy packs — the system has 15 element×variant packs.
  - Version stamps disagreed: `VERSION`/`package.json` at 1.0.0 vs `tokens/tokens.json`+`tokens.js` meta at 2.11.0 vs `tokens.dtcg.json` at 6.0.0.
- **Bilingual leaks:** `templates/email/Email.dc.html` footer ("Unsubscribe · Preferences") and `templates/slide-deck/SlideDeck.dc.html` ("Liquid Glass") stayed English in forced-VN mode; no template-level en/vi key-parity gate existed.
- **Component gaps:** `Dialog` missing body scroll-lock and `lang` absent from `Dialog.d.ts`; 3 components use a module-level custom `useId` (SSR duplicate-id risk); 8 components destructure a dead `children` prop; 23 DTCG leaves untyped (shadow/glass strings without `$type`).

### Medium (hygiene / maintainability)

- 5 export artifacts (~10 MB) tracked in `_audit/exports/` despite the gitignore policy saying they stay local.
- `templates/email/Email-VN-standalone.html` element enum listed 5 elements instead of 15.
- `templates/*/support.js` duplicated byte-identical ×84 (~5.6 MB total) — this is the Claude Design DC contract, so it stays, but its identity should be gated.
- `cx()` helper was copy-pasted across 98 component files (resolved: shared `components/_utils/cx.js`).
- `_audit/index.html` gate list lagged `run.html`.
- 233 MB local `examples/native/swiftui/.build` directory on disk (untracked, local-only noise).

## What the hardening change fixes vs defers

### Fixed in this change

- All doc-drift items above: README (suite count, Sources, component triplet wording, complete functional group lists, doc map incl. this file and `docs/quality-gates.md`, contrast-floor clarification), CONTRIBUTING scope claim, `docs/template-schema-v2.md` sidecar count, the three prompt files, and an llmstxt.org-format rewrite of `llms.txt`.
- Motion durations restored across `tokens.json` / `tokens.js` / `tokens.dtcg.json`; token metas aligned to the 1.0.0 pin; natives regenerated in lockstep.
- The two template EN leaks wrapped in proper en/vi holes; the stale standalone element enum fixed.
- CI: `test:unit` job added to the workflow; `_esm/` added to `package.json` `files`.
- `_audit/exports/*` untracked (files stay local; git now matches stated policy).
- New benchmark gates built and wired (see `docs/quality-gates.md` → "Planned gates"): token-format-parity, version-stamp, support-runtime-identity, package-exports-integrity, template-lang-parity, dtcg-typing — plus a stale-phrase blacklist extension to docs-consistency.
- In-repo deterministic esbuild bundle build + bundle-freshness gate (owner-approved C2), unblocking the JSX fixes: Dialog a11y (scroll lock, focus trap, `lang` in `Dialog.d.ts`), `React.useId` ×3, dead `children` removal ×8.
- Storybook filled out as the docs hub (Foundations/Guidelines/A11y/I18n/Status pages) and rebuilt.
- Root `DESIGN.md` generated from DTCG (Stitch open-spec format) + design-md-parity gate.

## How to re-run this audit

1. **Fast board:** serve the repo (`npx serve -l 8080 .`) and open `_audit/run.html`, or headless: `node _audit/ci/run-gates.mjs http://127.0.0.1:8080/_audit/run.html`. All hard gates must pass.
2. **Whole-set audits:** `_audit/responsive-overflow.html`, `_audit/language-overflow.html`, `_audit/theme-overflow.html` — every template × axis, minutes each (also run per PR + nightly in CI).
3. **Unit + provenance:** `npm run test:unit` and `node _audit/ci/check-token-provenance.mjs`.
4. **Cross-format token parity by hand:** compare any `tokens/*.css` value against `tokens.json` / `tokens.js` / `tokens.dtcg.json` / `tokens/native/*` (the token-format-parity gate now does this exhaustively).
5. **Doc-claim sweep:** `_audit/docs-consistency.html` for counts/pins, then grep for the retired phrases this audit removed (the stale suite count and the legacy pack count above) to prove none returned.
6. **Deep dives:** repeat the five-track method above; each track reads its whole surface, never a sample.
