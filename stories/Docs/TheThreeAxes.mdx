# The Three Axes

Every surface resolves three **independent** axes — the same three switches in this Storybook's toolbar (**Theme** · **Element** with all **15** packs · **Language**):

1. **Theme** — `data-theme` — light (default) · dark
2. **Element** — `data-cs-element` (+ optional `data-cs-variant`) — Ngũ Hành product identity: Kim · Mộc · Thủy · Hỏa · Thổ, **15 packs** in the Element toolbar (same set as **Foundations/Elements**, Identity Lab, and template EL maps)
3. **Language** — `lang` — `en` · `vi` (see **I18n**)

State them on a container; everything inside re-skins with no code change:

```html
<body data-theme="dark" data-cs-element="thuy" data-cs-variant="ocean" lang="vi">
  <!-- every component and template inside resolves all three axes -->
</body>
```

Defaults: `light · tho · en`. Surface treatment is liquid-glass — fixed, not an axis.

## Do

- Scope axes on a **container** and let the cascade work
- Consume `--cs-accent-*` for anything element-aware
- Ship every UI string through the EN·VN registry
- Test combinations that matter — axes multiply independently
- Keep dark-theme overrides in token space

## Don't

- Don't hardcode element hex values
- Don't put brand-umber text/borders on themeable surfaces without a dark override
- Don't remap semantic status colours or the 3px Ochre focus ring per element
- Don't mix two element accents as primary + secondary (gradient wash along Tương sinh only)
- Don't inline one-language strings in components
