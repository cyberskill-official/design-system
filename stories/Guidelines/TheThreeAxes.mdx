## Do

- Scope axes on a **container** (`body`, a template root, a preview wrapper) and let
  the cascade do the work.
- Consume `--cs-accent-*` roles for anything element-aware, so a product's element
  pack re-tints it automatically.
- Ship every UI string through the ENВ·VN registry so the Language axis actually works.
- Test all combinations that matter to you вЂ” the axes are independent, so light/dark Г—
  element Г— language multiply (this Storybook's toolbar exists for exactly that).
- Keep dark-theme overrides in token space (`[data-theme="dark"]` remaps token
  values); components should not branch on theme in code.

## Don't

- Don't hardcode element hex values вЂ” that breaks the Element axis (token names are
  the API).
- Don't put brand-umber text/borders on themeable surfaces without a dark override вЂ”
  umber is near-invisible on dark (gate-enforced by contrast-guard).
- Don't remap semantic status colours or the 3px Ochre focus ring per element вЂ” they
  are never elemental.
- Don't mix two element accents on one surface; a secondary element may appear only as
  a gradient endpoint along TЖ°ЖЎng sinh.
- Don't inline one-language strings in components вЂ” the Language axis must never
  strand a user.
