Bundle freshness gate

Browser replication of _audit/ci/check-bundle-freshness.mjs (the Node script stays the CI source of truth — it additionally discovers new source files by walking the tree, which a browser cannot). This page reads the /* @ds-bundle: … */ header off _ds_bundle.js line 1, fetches every recorded source file (currently 109), sha-256s each with SubtleCrypto, and asserts the first 12 hex chars equal the header's sourceHashes — i.e. the committed bundle was built from the component sources exactly as they exist now. Also cross-checks that every _ds_manifest.json component sourcePath appears in the header (compiler-known components can't silently miss the bundle). Any drift → rebuild with npm run build:bundle. Verdict on window.__bundlefresh.
