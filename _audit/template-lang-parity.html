Template language parity gate

Static source scan of all template .dc.html files (manifest-driven — currently 84). Three checks: (1) hole-map key parity — every template with const en = {…} / const vi = {…} maps must have identical, non-empty key sets; (2) branch pairing — every template using <sc-if value="{{ isEN }}"> must also carry the isVN branch (and vice versa), and every template exposing a language tweak must use at least one of the two mechanisms; (3) EN-leak lexicon — the email-chrome idioms that leaked pre-Phase-1 (Unsubscribe · View in browser · Preferences · All rights reserved · Manage subscription · Sent with) must not appear outside isEN-guarded regions or the en map literal. Documented scope exclusions (not gaps): the VN-first stacked instruments (37 vn-* + 3 doc-*, no language tweak) render Vietnamese and English together by design, so a hidden-language "leak" cannot exist there — they are exempt from check 3 (checks 1–2 still apply if they ever grow maps/branches); brand/product names that legitimately appear in Vietnamese copy (e.g. Status Hub, CyberSkill) are deliberately not in the lexicon. Verdict on window.__langparity.
